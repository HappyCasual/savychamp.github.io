Automating 2D Game Assets: A Python Pipeline

Published in Developer Logs • SavyChamp Studio

As a solo indie developer, time is the most valuable resource. When building complex 2D puzzle games like Jig Cards or Love Merge, managing the sheer volume of visual assets can quickly become overwhelming. Every icon, UI button, and puzzle piece needs to be perfectly formatted, sized, and organized. If handled manually, this process can eat up hundreds of hours that would be better spent writing logic code or designing new levels.

To combat this, we rely heavily on Python to automate our art pipeline. By writing custom scripts, we can turn tedious, multi-hour organizational tasks into a single-click process.

The Duplicate Asset Problem

When prototyping a new game, it is easy for project folders to become cluttered. You might have three different folders containing various iterations of a "glossy candy-style" UI button, making it difficult to know which one is the final version. Manually clicking through hundreds of PNG files to check for duplicates is incredibly inefficient.

To solve this, we developed a lightweight Python GUI tool. At runtime, the developer can select three or more root directories. The script recursively scans all sub-folders, generates a unique hash for every single image file, and instantly flags any exact duplicates across the entire project structure. This ensures our final game build only contains the necessary files, keeping the overall app size as small as possible.

High-Quality SVG Conversions

Another major bottleneck is vector conversion. In modern mobile development, scaling graphics cleanly across different screen sizes is mandatory. While raster images (like PNGs) are great for complex art, flat UI elements are often better served as scalable vector graphics (SVGs).

Instead of manually opening illustrator software for every single file, we utilize Python scripts to handle batch conversions. By passing high-resolution images through a programmatic tracer, we can generate incredibly clean SVGs automatically. This ensures that when a user plays our games on a high-density tablet display, the UI remains perfectly crisp without pixelation.

Conclusion

Programming isn't just about writing the game itself; it is about building the tools that help you make the game faster. By utilizing Python to automate the boring parts of asset management, SavyChamp can maintain a rapid development pace while still delivering premium, highly polished visual experiences to our players.
